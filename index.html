핵심 요구사항 분석

  • 키워드 관리 — 신규 등록, 진행 상태(대기/작성중/검수/발행완료/순찰), 우선순위·담당자 배정, 메인-세부 키워드 트리 묶음, 발행 게시글 매핑
  • 원고 관리 — 리치 텍스트 에디터, 다중 이미지 첨부, 네이버 카페·블로그 발행 전 미리보기, GPT/Claude API 기반 초안 생성·윤문 (API Key 입력형)
  • 발행 자동화 — 네이버 카페·블로그 자동 발행, 예약 발행 스케줄러, 다계정 등록 및 라운드로빈/조건부 선택, 성공·실패 사유 로그
  • 순찰/크롤링 — 발행글 노출 누락 자동 체크, 원고 수정 이력, 댓글 작업 완료 체크, 누락 시 재발행 플래그 및 상태 추적
  • 시스템 연동·관리 — Google Sheets API Import/Export, 직원별 RBAC 권한, 전 작업 감사 로그, 운영 현황 대시보드
  • 비기능 요구사항 — Sheets 할당량 회피를 위한 자체 PostgreSQL 메인 사용, 네이버 봇 탐지(CAPTCHA/IP 차단) 리스크 최소화 아키텍처, 반자동화 옵션 포함

제안 포인트

1. 콘텐츠 운영 전 워크플로우의 단일 어드민 통합. 현재 분산된 스프레드시트(키워드/원고/발행 현황)를 단일 도메인 모델로 재정의하고, 키워드 → 원고 → 발행 → 순찰까지 상태 머신 기반으로 추적합니다. 어느 단계에서 누가, 무엇을, 언제 처리 중인지 한 화면에서 파악 가능한 운영 환경을 제공합니다.

2. 자체 PostgreSQL DB 중심 + Google Sheets 보조 채널 구조. Google Sheets API의 할당량(분당 호출 제한, 셀 5백만 한도) 문제를 회피하기 위해 자체 DB를 1차 소스로 두고, Sheets는 기존 데이터의 일괄 Import / 보고 목적의 정기 Export 채널로만 활용합니다. 운영 데이터가 100만 건 이상으로 늘어나도 안정적으로 동작합니다.

3. 반자동화 모드 기본 채택 — 계정 차단 리스크 최소화. 네이버는 카페·블로그 발행을 위한 공식 API를 제공하지 않으므로 Playwright 헤드리스 브라우저 자동화가 불가피합니다. 100% 무인 자동화는 캡차 발생·계정 차단 리스크가 크므로, 휴먼-라이크 입력 패턴 + 다계정 라운드로빈 + 캡차/이상 감지 시 운영자 알림(수동 개입)을 결합한 반자동화 모드를 기본 옵션으로 제공합니다. 차단 발생 시 영향 범위를 단일 계정으로 격리합니다.

4. GPT/Claude API Key 입력형 — 비용·보안 클라이언트 통제. AI 윤문/초안 생성은 클라이언트가 보유한 OpenAI 또는 Anthropic API Key를 입력해 사용하는 방식으로 설계합니다. 시스템은 호출 횟수/실패만 기록하고, Key는 AES-256으로 사용자 단위 분리 저장합니다. AI 사용 비용은 클라이언트가 직접 통제하고, 약정 사용량 제한도 자유롭게 조정할 수 있습니다.

5. 마일스톤별 PoC 우선 — Day 14 발행 PoC 실증. 자동화 프로젝트의 가장 큰 리스크는 네이버 자동 로그인·발행이 환경에 따라 실패하는 사례입니다. 기획·설계 단계와 병행하여 Day 14까지 네이버 카페·블로그 1건 자동 발행 PoC를 실증하고, 결과에 따라 반자동화 비율을 조정합니다. 본격 개발 전 기술 리스크를 검증합니다.

6. 관리자 어드민·리치 에디터·워크플로우 경험. 50+ 화면 규모의 B2B 어드민, Lexical 기반 리치 텍스트 에디터(170 파일 / 26 커스텀 노드), 다단계 결재·승인 워크플로우, AI 보고서 자동 생성(ChatGPT 연동) 등 본 프로젝트의 핵심 기능과 직결되는 유사 프로젝트를 다수 수행했습니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

현재 클라이언트는 키워드, 원고, 발행 현황 등 콘텐츠 운영 데이터를 여러 스프레드시트로 분산 관리하고 있습니다. 운영 인력이 늘어남에 따라 데이터 동기화 누락, 중복 작업, 상태 추적 곤란 등의 문제가 누적되고 있으며, Google Sheets API 할당량 한계로 자동화 확장도 어려운 상황입니다.

본 프로젝트는 키워드 등록부터 원고 작성, 네이버 자동 발행, 노출 순찰, 수정 관리까지의 전 과정을 단일 웹 기반 어드민으로 통합하여 운영 효율을 극대화하는 것을 목표로 합니다. 자체 PostgreSQL DB를 중심에 두고, Google Sheets는 Import/Export 보조 채널로 활용합니다.

3. 기능 요구사항

3.1 키워드 관리

  • [필수] 키워드 등록(단일/대량 업로드) 및 진행 상태(대기/작성중/검수/발행완료/순찰완료) 관리
  • [필수] 키워드별 우선순위(High/Mid/Low), 담당자, 마감일 배정
  • [필수] 메인 키워드 → 세부 키워드 트리 묶음 관리
  • [필수] 해당 키워드로 발행된 게시글 매핑 및 발행 유무 표시
  • [선택] 키워드 검색량/경쟁도 메타 데이터 필드 (수동 입력)

3.2 원고 관리

  • [필수] 리치 텍스트 에디터(서식, 표, 인용, 코드, 링크) 및 다중 이미지 첨부

5. 기술 제약사항

  • 네이버는 카페·블로그 발행을 위한 공식 자동화 API를 제공하지 않으므로, Playwright 기반 브라우저 자동화 사용이 불가피합니다.
  • 100% 무인 자동화는 계정 차단 리스크가 존재하므로, 캡차 발생 / 비정상 차단 감지 시 운영자에게 알림을 보내 수동 개입을 받는 반자동화 모드를 기본값으로 제안합니다.
  • OpenAI / Anthropic API 사용 비용은 클라이언트 API Key 입력 방식으로 운영되어, 시스템은 호출 횟수·실패만 기록합니다.
  • IP 풀(프록시) 및 다계정 운영은 클라이언트가 제공하는 자원에 따라 활용 수준이 결정됩니다.
  • 네이버 정책 변경에 따라 자동화 로직 재구축이 필요할 수 있으며, 이는 본 계약 범위 외 별도 협의 사항입니다.
